Keyterm Models

Keyterm models create keyterms for text sent to table endpoints:

    {% for model in models %} {% if model.kind == 'keyterms' %}
  • {{ model.name }}
    • {% endif %} {% endfor %}

Keyterm extraction is used by FeatureBaseAI in conjunction with embeddings to create a balanced graph of entries in FeatureBase using a technique called SlothANN. Because FeatureBase can scale to millions or billions of entries, these graphs can be used to segment the vector space for efficient similarity searches.

sloth-extract: A FeatureBase ensemble of MiniLM, H2-keywordextractor, BERT and Fasttext models.

Other Models

In the near term, FeatureBaseAI is planning on supporting many different types of models for processing text and other data sent to its endpoints. Templates will become available for models added to the system, which will likely include:

  • CodeLLaMa Instruct
  • LLaMa 2
  • Claude v2
  • OpenAI's GPT
  • Bard

Models may be considered a union of templates, data storage for prompting, and a specific trained model. Additionally, FeatureBaseAI will also provide support for training models in the near future, using datasets created by training the models on the data ingested previously.
